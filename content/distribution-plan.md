# 콘텐츠 배포 전략 — "1인 AI 에이전시 만들기"

> 목적: 개인 브랜드/권위 구축 → 장기적으로 에이전시·SaaS 퍼널 자산화
> 작성: 2026-06-17

---

## 1. 타겟 (WHO)

### Primary — 한국어권 1인 빌더 / AI 실전 적용층
- 25~40세, 사이드프로젝트·독립·"AI로 뭔가 만들어 팔고 싶다"는 욕구
- AI 도구는 써봤지만 "그냥 만들면 안 되네"를 느낀 사람 → thesis("판단 > 도구")가 가장 깊게 꽂히는 층
- 빌드인퍼블릭(build-in-public) 콘텐츠를 즐겨 봄
- **이들이 곧 미래 고객·추천인·동료다.** 권위가 여기서 쌓이면 인바운드가 따라옴.

### Secondary — 소상공인 / 클리닉 오너 (잠재 고객 인접)
- 직접 타겟팅하지 않음. Primary층 권위가 흘러들어가는 2차 효과로만 노림.

### 안 할 것
- 처음부터 "고객 유치"를 노리고 영업톤으로 쓰지 않음 (권위가 먼저, 영업은 결과)

---

## 2. 플랫폼 스택 (WHERE) — 3채널 집중

여러 곳에 얇게 뿌리지 말고 역할별 3채널에 집중.

| 역할 | 채널 | 왜 |
|------|------|-----|
| **원본 + 구독 (canonical)** | **브런치** | 한국 스토리텔링 독자, 다음/카카오 검색 노출, 구독 기능 내장, 셋업 빠름. 케이스 스터디 장르에 최적. |
| **메이커 권위 허브** | **디스콰이엇 (disquiet.io)** | 한국 메이커 커뮤니티의 본진. "1인 AI 에이전시 만들기"를 메이커로그로 등록하고 매 편 업데이트 → 빌더 권위의 핵심. |
| **깔때기 상단 (단문)** | **스레드(Threads)** | 한국에서 빌드인퍼블릭 단문에 가장 강함. 글당 인사이트 3~5개를 원자화해 데일리 포스팅, 원본으로 유입. |

**선택(여력 되면) — 링크드인:** B2B·프로페셔널 권위 + 시드니 로컬/글로벌 빌더. 영문 병행 시 reach 확장. 초반엔 생략, 5편쯤 쌓이면 검토.

**구독 자산 욕심나면:** 브런치 대신/병행으로 **메일리(Maily)** 또는 **Stibee**. 이메일은 소유한 청중이라 시리즈 모델에 강함. 단 초반 셋업 부담 → 글 5~10편 쌓인 뒤 이전 권장.

## 3. 채널별 포맷 변환 규칙 (한 번 쓰고 N번 쓴다)

| 채널 | 길이/포맷 | 톤 |
|------|----------|-----|
| 브런치 | 풀 케이스 스터디 (1,500~2,500자) | 서사 + 판단 드러내기 |
| 디스콰이엇 | 로그 업데이트 (3~5문단) + 원본 링크 | "이번 편에서 배운 것" 요약, 솔직한 빌더 톤 |
| 스레드 | 1훅 + 2~4 인사이트 단문 (각 ~300자) | 짧고 날카롭게, 한 포스트=한 인사이트 |
| (링크드인) | 800~1,200자 + 비주얼 1장 | 프로페셔널, "업종 맥락 이해" 강조 |

## 4. 발행 시퀀스 — 글 1편 = 1주일치 콘텐츠

| 시점 | 액션 |
|------|------|
| **Day 0** | 브런치 발행 (canonical) + 디스콰이엇 메이커로그 업데이트(+링크) |
| **Day 1** | 스레드: 글의 훅 1개 (thesis 단문) + 원본 링크 |
| **Day 2** | 스레드: 인사이트 ① (미니 스토리 형태) |
| **Day 3** | 스레드: 인사이트 ② |
| **Day 5** | 스레드: 반응 좋았던 단문 재구성 / 카드 이미지 1장 |
| **Day 7** | 스레드: 회고 단문("이 글 쓰고 배운 것") + 다음 편 예고 |

→ 장문은 주 1편, 단문은 거의 매일. 제작 부담은 낮고 노출은 매일.

## 5. 30일 캘린더

| 주차 | 장문 (브런치/디스콰이엇) | 단문 (스레드, 데일리) |
|------|--------------------------|------------------------|
| **W1** | 0편 발행 (pillar: 판단>도구) | 0편 인사이트 원자화 |
| **W2** | 1편 발행 (심리상담: 후기 없이 신뢰) | 1편 인사이트 원자화 |
| **W3** | 2편 작성·발행 (WeSkin: 번역≠세그먼트) | 2편 인사이트 원자화 |
| **W4** | 3편 작성·발행 (Sharna: 피드백 루프 도구화) + 시리즈 회고글 | 가장 반응 좋은 단문 재활용 |

발행 요일 고정 추천: **장문 = 화요일 오전, 회고 = 일요일 저녁** (한국 시간 기준 도달률 양호 구간).

## 6. 측정 지표 & 의사결정 룰

**북극성 지표:** 구독/팔로우 증가수 (조회수보다 우선 — 소유 청중이 자산).

추적:
- 브런치 구독자 / 디스콰이엇 부스트·댓글 / 스레드 팔로워·저장
- 단문별 반응 (어떤 인사이트가 먹히는지 = **다음 장문 소재의 신호**)

의사결정 룰:
1. 스레드 단문 중 반응 top → 그 주제를 다음 장문에서 심화.
2. 4주 후 채널별 ROI 비교 → 가장 약한 1채널 버리고 강한 곳에 집중.
3. "고객 문의"가 들어오기 시작하면 → 별도 랜딩/포트폴리오 페이지로 분기 (영업 모드 전환).

## 분기점 (전략이 갈리는 지점)

- **만약 1순위가 "시드니 로컬 클리닉 직접 공략"이라면** → 플랫폼을 인스타/네이버블로그/로컬 커뮤니티로 교체하고 톤을 영업형으로. 현재 계획은 "빌더 권위 → 인바운드" 경로 가정.
- **글로벌 reach까지 원하면** → 영문 병행(Substack + X + LinkedIn). 단 번역 부담 ↑, 초반엔 한국어 집중 권장.
